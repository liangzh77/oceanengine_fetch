# 巨量引擎数据抓取工具 - 开发计划

## 一、技术架构设计

### 1.1 整体架构

两个独立的命令行工具，由外部定时任务调度：

```
外部定时任务（每半小时调用）
    │
    ├──> 工具1: fetch_data.py（抓取数据）
    │    ├─ Playwright 打开浏览器
    │    ├─ 切换3个组织，抓取表格
    │    └─ 数据存入 SQLite
    │
    └──> 工具2: check_rules.py（规则检查）
         ├─ 读取最新一批抓取数据
         ├─ 逐条规则匹配
         └─ 命中则飞书通知
```

### 1.2 技术栈

| 模块 | 技术选型 | 理由 |
|------|---------|------|
| 浏览器自动化 | Playwright | 支持持久化上下文，登录状态保存方便 |
| 数据库 | SQLite | 轻量级，无需额外部署 |
| 通知 | 飞书 Webhook | 官方API，稳定可靠 |
| 配置管理 | YAML | 人类可读，易于维护 |
| 日志 | logging | 标准库 |

## 二、数据库设计

### 2.1 表结构

#### organizations（组织表）
```sql
CREATE TABLE organizations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,        -- 组织名称（天津乾飞/北京奥吟）
    app_name TEXT NOT NULL,    -- 应用名称（咕友app/丝话app/缘话app）
    UNIQUE(name, app_name)
);
```

#### accounts（账户表）
```sql
CREATE TABLE accounts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    org_id INTEGER NOT NULL,
    account_name TEXT NOT NULL,
    account_budget REAL,
    cost REAL,
    daily_roi REAL,
    fetch_time TIMESTAMP NOT NULL,
    FOREIGN KEY (org_id) REFERENCES organizations(id)
);
CREATE INDEX idx_accounts_fetch_time ON accounts(fetch_time);
```

#### projects（项目表）
```sql
CREATE TABLE projects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    account_id INTEGER NOT NULL,
    project_name TEXT NOT NULL,
    project_budget REAL,
    cost REAL,
    daily_roi REAL,
    status TEXT,
    bid_price REAL,
    fetch_time TIMESTAMP NOT NULL,
    FOREIGN KEY (account_id) REFERENCES accounts(id)
);
CREATE INDEX idx_projects_fetch_time ON projects(fetch_time);
```

#### units（单元表）
```sql
CREATE TABLE units (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    unit_name TEXT NOT NULL,
    cost REAL,
    daily_roi REAL,
    status TEXT,
    fetch_time TIMESTAMP NOT NULL,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
CREATE INDEX idx_units_fetch_time ON units(fetch_time);
```

#### rules（规则表）
```sql
CREATE TABLE rules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    description TEXT,
    target_type TEXT NOT NULL,       -- 'account' / 'project' / 'unit'
    condition_field TEXT NOT NULL,   -- 'cost' / 'daily_roi' / 'status' 等
    operator TEXT NOT NULL,          -- '>' / '<' / '>=' / '<=' / '==' / '!='
    threshold_value TEXT,            -- 阈值（TEXT类型兼容数字和字符串比较）
    enabled BOOLEAN DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### notifications（通知记录表）
```sql
CREATE TABLE notifications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    rule_id INTEGER NOT NULL,
    target_type TEXT NOT NULL,
    target_name TEXT NOT NULL,
    message TEXT NOT NULL,
    sent_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (rule_id) REFERENCES rules(id)
);
```

### 2.2 数据关系
```
organizations (1) ──> (N) accounts
accounts (1) ──> (N) projects
projects (1) ──> (N) units
rules (1) ──> (N) notifications
```

## 五、工具调用方式

```bash
# 工具1：抓取数据（首次需人工登录）
python fetch_data.py                  # 抓取所有组织数据
python fetch_data.py --login          # 强制重新登录

# 工具2：规则检查
python check_rules.py                 # 检查最新数据并通知

# 规则管理
python check_rules.py --add "ROI过低" --target project --field daily_roi --op "<" --value 1.5
python check_rules.py --list          # 列出所有规则
python check_rules.py --delete 3      # 删除规则
python check_rules.py --enable 3      # 启用规则
python check_rules.py --disable 3     # 禁用规则

# 验证脚本
python verify_scraper.py              # 验证抓取能力
```

## 六、风险与应对

| 风险 | 应对措施 |
|------|---------|
| 网页结构变化 | 灵活选择器 + 容错逻辑 |
| 登录状态失效 | 检测失效后飞书通知，需人工重新登录 |
| 网络不稳定 | 重试机制 + 失败日志 |
